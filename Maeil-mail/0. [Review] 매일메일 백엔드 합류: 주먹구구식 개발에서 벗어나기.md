### 1. 익숙한 환경에서의 정체

지금까지 내가 서버 개발자로서 제대로 된 코드를 작성해 본 경험은 'moA' 프로젝트가 전부였다. 당시에는 프로젝트를 어떻게 구성해야 할지 몰라 친구가 세팅해둔 환경이나 좋다고 추천하는 방식들을 무작정 따랐다. 친한 친구들끼리 진행하다 보니 서로의 코드를 꼼꼼히 리뷰하기보다는 "일단 돌아가니까"라는 식으로 머지하는 경우가 많았다.

시간이 지나면서 불안함이 생겼다. 만약 내가 회사에 취업해서 아예 모르는 사람들과 작성된 코드를 마주했을 때, 과연 내가 1인분의 몫을 할 수 있을지에 대한 의구심이었다. 내가 처음부터 세팅하지 않은 프로젝트에서 코드를 이해하고 작업하는 능력이 스스로에게도 부족하게 느껴졌다. 그래서 조금 더 낯설고 엄격한 환경에서 나를 시험해보고 싶었다.

### 2. 매일메일 팀으로 보낸 메일과 답장

교내 프로젝트 합류를 고민하던 중, 평소 즐겨 사용하던 서비스인 '매일메일' 팀에 합류 가능 여부를 묻는 메일을 보냈다. 사실 확신보다는 "안 되면 말고"라는 마음이 컸다. 그리고 며칠 뒤 팀 리더 하늘님으로부터 답장을 받았다.

### 3. 멀티 모듈과 낯선 코드 분석

합류 후 2주 동안은 기존 프로젝트의 코드를 분석하는 데 집중했다. 가장 먼저 마주한 기술적인 벽은 멀티 모듈 구조였다. 이전까지 단일 모듈 형태의 간단한 프로젝트만 진행해왔던 나에게, 계층별로 분리된 모듈 간의 의존 관계는 꽤나 생소했다.

활성 사용자가 많은 서비스라는 점도 큰 부담이었다. 내가 짠 코드 한 줄이 서비스 운영에 직접적인 영향을 미칠 수 있다는 사실이 압박으로 다가왔다. 하지만 그만큼 배울 것이 많을 것이라는 확신이 들었다.

</br>


### 4. 온보딩 미팅과 깨달음

방금 온보딩 느낌의 미팅을 진행하며 하늘님과 깊은 대화를 나누었다. 하늘님은 나에게 "이 서비스에서 무엇을 얻고 싶은지"를 물으셨다.

**Q. 이 서비스에서 얻고 싶은 게 무엇인가요?**

A. 학부 수준의 프로젝트를 넘어 현업 수준의 코드를 경험하고 싶습니다. 특히 제가 평소에 애용하는 서비스의 코드에 직접 기여할 기회를 얻었다는 것만으로도 정말 감사합니다.

**Q. 활성 사용자가 많은 게 왜 본인에게 중요한가요?**

A. "그동안은 친구들과 편하게 작업하다 보니 코드를 꼼꼼히 보지 않고 머지하는 경우가 많았습니다. 하지만 실제 사용자가 많은 환경에서는 내 코드 한 줄이 미치는 영향력이 크다는 것을 압니다. 그 '압박감'을 스스로에게 주어, 더 꼼꼼하게 컨벤션을 지키고 공부하며 개발하고 싶기 때문입니다.

실력과 경험이 부족한 나에게 기회를 주신 것에 대해 여러 번 감사 인사를 전했다. 단순히 사용자가 많아서 좋다는 대답보다, 그 환경이 주는 압박을 성장의 동력으로 삼고 싶다는 내 진심이 잘 전달되었기를 바란다.

</br>


### 5. 앞으로의 기여: 작지만 단단한 시작

당장은 대단한 기능을 추가하기보다 리팩토링 요소가 적더라도 누락된 테스트 케이스를 채우는 일부터 시작할 계획이다. 코드를 하나하나 뜯어보며 이해되지 않는 부분들을 찾아가는 과정이 나에게는 가장 큰 공부가 될 것 같다. 이후에는 유료화 전환이나 결제 프로세스 도입 같은 굵직한 과제들을 경험하며 서버 개발자로서의 역량을 키워보고 싶다.

익숙함을 벗어나 낯선 코드 속에서 길을 찾는 과정은 분명 고통스럽겠지만, 이 과정을 마쳤을 때 나는 한 단계 더 단단해진 개발자가 되어 있을 것이라 믿는다. 이제 시작이다.
